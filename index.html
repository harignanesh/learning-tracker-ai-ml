Phase 1: Generative AI Foundations
Understanding the engine before driving it
0%
▼
What is this? Before writing a single line of code, you need to understand how AI thinks. This phase teaches you what generative AI is, how it works under the hood, and why it produces the outputs it does. Think of it like learning how a car engine works before becoming a driver.

Why does it matter? Without this, you will blindly copy-paste AI code without knowing why it fails or how to fix it.
Core Topics
✓
What is Generative AI (3 subtopics)
AI that creates new content (text, images, code) instead of just classifying existing content. Like the difference between a judge scoring a drawing vs. an artist making one.
concept
✓
Discriminative vs Generative Models
Discriminative: 'Is this a cat?' → Yes/No. Generative: 'Make me a cat image.' Very different goals.
✓
History: From Rule-based AI to LLMs
AI went from hand-coded rules (1950s) → machine learning (1990s) → deep learning (2010s) → transformers (2017) → ChatGPT era (2022+).
✓
Current Landscape: GPT, Gemini, Claude, Llama
The big players and their strengths. OpenAI (GPT-4o), Google (Gemini), Anthropic (Claude), Meta (Llama - open source).
✓
Neural Networks Basics (4 subtopics)
The building blocks of all modern AI. A network of math functions loosely inspired by how brain neurons fire. Input → process → output.
concept
✓
Neurons, Weights, and Biases
Each neuron takes inputs, multiplies by weights (importance scores), adds a bias (offset), and passes to an activation function.
✓
Forward Pass and Backpropagation
Forward: compute prediction. Backward: measure how wrong it was, then adjust weights to be less wrong. Repeat millions of times.
✓
Activation Functions (ReLU, Sigmoid, Softmax)
ReLU: if negative → 0, else keep. Sigmoid: squash to 0-1 (good for yes/no). Softmax: convert scores to probabilities that sum to 1.
✓
Loss Functions and Gradient Descent
Loss = how wrong the model is. Gradient descent = the algorithm that walks downhill on the loss landscape to minimize errors.
▶ Try: Neural Network Playground
✓
Transformers Architecture (5 subtopics)
The revolutionary architecture (2017) that powers every major AI today. Key idea: instead of reading text one word at a time, pay attention to ALL words at once.
concept
✓
The Attention Mechanism (simply explained)
For each word, ask: which other words are most relevant to understanding this word? 'The bank was steep' — 'bank' should attend to 'steep', not 'money'.
✓
Multi-head Self-Attention
Run attention multiple times in parallel, each head learning different relationships (syntax, semantics, coreference, etc).
✓
Encoder vs Decoder vs Encoder-Decoder
Encoder: reads and understands (BERT). Decoder: generates text (GPT). Encoder-Decoder: translate/summarize (T5, BART).
✓
Positional Encoding
Transformers process all words at once, so they need a way to know word order. Positional encoding adds position information.
✓
Layer Normalization and Feed-Forward Layers
After attention, data passes through normalization (stabilize training) and a feed-forward network (add non-linearity).
▶ Visualize Attention
✓
Large Language Models (LLMs) (4 subtopics)
A transformer trained on massive text data to predict the next word. Do this billions of times and it learns grammar, facts, reasoning, and coding.
concept
✓
Pre-training vs Fine-tuning vs RLHF
Pre-training: predict next word on internet text. Fine-tuning: specialize on task-specific data. RLHF: humans rate outputs to make it helpful and safe.
✓
Parameters and Scale Laws
Parameters = knobs in the model (GPT-4 has ~1.7 trillion). More parameters + more data + more compute = smarter model (Scaling Laws).
✓
Emergent Abilities
At certain scales, models suddenly gain abilities nobody explicitly trained — like multi-step math, code generation, theory of mind. This surprises even researchers.
✓
Model Families: GPT-4, Claude 3, Gemini 1.5, Llama 3
Each family has different strengths. Claude excels at safety/reasoning. GPT-4 at breadth. Gemini at multimodal. Llama at open-source customization.
▶ Try LLMs Free: Google AI Studio
✓
Tokens and Context Windows (4 subtopics)
AI doesn't read words — it reads tokens (chunks of ~4 characters). The context window is how many tokens it can 'see' at once — its working memory.
concept
✓
What is a Token?
'ChatGPT is great' = 4 tokens. One word ≈ 0.75 tokens. 1000 tokens ≈ 750 words. Knowing this helps you estimate costs and limits.
✓
Tokenization Algorithms (BPE, WordPiece)
BPE (Byte Pair Encoding): merge frequent character pairs into tokens. Used by GPT. WordPiece: similar, used by BERT.
✓
Context Window Limits and Long Context Models
GPT-4: 128k tokens. Claude 3: 200k tokens. Gemini 1.5: 1M tokens. Longer = can read full books, but also costs more.
✓
Attention and Context: Why Position Matters
LLMs often pay less attention to the middle of a long context ('lost in the middle'). Important info should be at start or end.
▶ Count Tokens: OpenAI Tokenizer
✓
Sampling Parameters (5 subtopics)
The knobs that control how creative vs predictable the AI is. Temperature = randomness dial. top-p = how many words to consider.
concept
✓
Temperature (0 to 2)
0 = always picks most likely word (robotic, deterministic). 1 = normal. 2 = very creative/chaotic. For code: use 0.2. For stories: use 0.8-1.2.
✓
Top-p (Nucleus Sampling)
Instead of using all vocabulary, only sample from the top words whose probabilities sum to p. top-p=0.9 cuts out rare words.
✓
Top-k Sampling
Only consider the top-k most likely next words. top-k=50 means choose from 50 candidates only.
✓
Frequency and Presence Penalties
Frequency penalty: reduce words that appeared often (avoids repetition). Presence penalty: reduce any word that appeared (encourages diversity).
✓
max_tokens and stop sequences
max_tokens caps output length. Stop sequences tell the model to stop generating when it hits a specific string like '###' or 'END'.

Phase 2: Prompt Engineering
The art of talking to AI effectively
0%
▼
What is this? Prompt engineering is writing instructions to AI that get reliable, high-quality results. It's like knowing how to ask a question properly — a vague question gets a vague answer.

Why does it matter? 80% of AI failures are prompt failures. A good prompt engineer can make a weak model perform like a strong one.
Core Topics
✓
Prompt Structure and Anatomy (4 subtopics)
Every great prompt has a system role, context, task, format, and examples. Missing any of these causes vague outputs.
concept
✓
System Prompt vs User Prompt
System: the AI's persistent role/persona ('You are a senior Python developer...'). User: the actual task. System sets the stage; user asks the question.
✓
Instruction Clarity and Specificity
Bad: 'Write an email'. Good: 'Write a 100-word professional email to a client explaining a 2-day delay in project delivery. Tone: apologetic but confident.'
✓
Output Format Control (JSON, Markdown, Lists)
Tell AI exactly what format you want: 'Respond ONLY as a JSON object with keys: name, age, summary. No preamble.' This prevents hallucinated structure.
✓
Role Prompting
'You are a world-class data scientist reviewing my Python code. Be critical and specific.' Role prompting unlocks specialized vocabulary and reasoning.
▶ Practice on Claude
✓
Few-Shot Prompting (3 subtopics)
Show the AI examples of what you want before asking. 1 example = one-shot. 2+ examples = few-shot. AI learns the pattern instantly.
concept
✓
Zero-shot vs One-shot vs Few-shot
Zero-shot: no examples (relies on training). One-shot: 1 example. Few-shot: 2-5 examples. Few-shot dramatically improves consistency.
✓
Choosing Good Examples
Examples should cover edge cases, be diverse, and match the exact format you want in the output. Bad examples produce bad outputs.
✓
In-Context Learning
LLMs can learn a new task purely from examples in the prompt — without any weight updates. This is called in-context learning and is unique to large models.
▶ OpenAI Playground
✓
Chain-of-Thought Prompting (4 subtopics)
Tell the AI to think step by step before answering. This dramatically improves reasoning on math, logic, and multi-step problems.
concept
✓
CoT: Just add 'Think step by step'
Simply adding 'Let's think step by step' before the answer increases accuracy on math benchmarks by 40-50%. It forces the model to reason aloud.
✓
Zero-Shot CoT vs Few-Shot CoT
Zero-Shot CoT: just add the magic phrase. Few-Shot CoT: provide full reasoning examples. Few-shot CoT works better for hard problems.
✓
Self-Consistency: Sample Multiple Reasoning Paths
Run the same CoT prompt 5-10 times, then take the majority answer. Dramatically improves accuracy on complex problems.
✓
Tree of Thought (ToT)
Instead of one chain of reasoning, explore multiple branches simultaneously (like a decision tree) and evaluate which path leads to the best answer.
▶ Try on ChatGPT
✓
Advanced Prompting Techniques (5 subtopics)
Beyond basics — techniques used by professionals to get consistent, reliable, production-ready outputs from AI.
concept
✓
ReAct (Reason + Act)
Alternate between reasoning ('I need to find X') and acting (calling a tool). Used heavily in AI agents.
✓
Prompt Chaining
Break a complex task into multiple prompts where output of one feeds input to next. Better than one massive prompt for complex workflows.
✓
Delimiters and Injection Prevention
Use XML tags (, ) or triple backticks to clearly separate parts of your prompt. Prevents prompt injection attacks.
✓
Constitutional AI and Self-Critique
Ask the AI to critique its own answer against a set of rules, then rewrite. Dramatically improves quality without human review.
✓
Meta-Prompting
Ask the AI to write or improve the prompt itself: 'Here is my prompt. Improve it to get better results for this task.' AI is surprisingly good at this.
▶ Anthropic Prompt Library
✓
Prompt Evaluation and Testing (4 subtopics)
Prompts fail in unexpected ways. You need a system to test, measure, and improve prompts systematically — not just vibe-check them.
concept
✓
Evaluation Criteria: Correctness, Format, Safety
Define what 'good' means before testing. Correctness (right answer?), format (right structure?), tone (right voice?), safety (no harmful output?).
✓
Building a Test Set
Create 20-50 test inputs with expected outputs. Every prompt change should improve scores on your test set, not just on one example.
✓
LLM-as-Judge
Use a strong LLM (GPT-4, Claude) to evaluate outputs of a weaker model. Give it a rubric. This enables automated evaluation at scale.
✓
Prompt Versioning and A/B Testing
Treat prompts like code. Version control them. A/B test variants on real traffic. Track which version performs better.

Phase 3: Embeddings & Semantic Search
Teaching AI to understand meaning, not just keywords
0%
▼
What is this? Embeddings convert text into lists of numbers (vectors) that capture meaning. Similar meaning = similar numbers = close together in space. This enables search by meaning, not just keywords.

Why does it matter? Search engines use keywords. Semantic search understands 'heart attack' and 'myocardial infarction' are the same thing. This is the backbone of RAG systems.
Core Topics
✓
What are Embeddings? (3 subtopics)
A way to represent text as a list of 768-3072 numbers where similar meaning = similar numbers. 'Dog' and 'puppy' will be very close. 'Dog' and 'Python (language)' will be far apart.
concept
✓
Word Embeddings vs Sentence Embeddings
Word: one vector per word (Word2Vec, GloVe). Sentence: one vector per sentence/paragraph (BERT, OpenAI text-embedding-3). Sentence embeddings are used in modern RAG.
✓
Embedding Dimensions Explained
A 1536-dimensional embedding = a list of 1536 numbers. Each dimension captures a different semantic feature. You don't need to understand each — the model learns them.
✓
Popular Embedding Models (OpenAI, Cohere, BGE)
OpenAI text-embedding-3-small (1536d, cheap). Cohere embed-v3 (great for multilingual). BGE-M3 (open source, best free option). Sentence-BERT for local use.
▶ Visualize Embeddings: Embedding Projector
✓
Vector Similarity (4 subtopics)
Once you have vectors, you need to measure how close two vectors are. This tells you how similar two pieces of text are in meaning.
concept
✓
Cosine Similarity (most important)
Measures the angle between two vectors. 1 = identical direction (same meaning). 0 = perpendicular (unrelated). -1 = opposite. Range: -1 to 1.
✓
Euclidean Distance
Straight-line distance between two points in high-dimensional space. Simple but less effective than cosine for text embeddings.
✓
Dot Product
Similar to cosine similarity but affected by vector magnitude. Used internally in attention mechanisms. Fast to compute.
✓
When to Use Which Metric
Cosine: best for text semantic similarity. Euclidean: best for image/audio. Dot product: when vectors are normalized (same as cosine).
▶ Try Cosine Similarity Online
✓
Semantic Search System (4 subtopics)
Encode all your documents into vectors, store them, then encode a query and find the nearest vectors. Those are your most relevant documents.
concept
✓
The Encode-Store-Query Pipeline
Step 1: Embed all documents. Step 2: Store vectors. Step 3: Embed query. Step 4: Find top-K nearest vectors. Step 5: Return original documents.
✓
Approximate Nearest Neighbor (ANN) Search
Exact nearest neighbor is too slow for millions of vectors. ANN finds near-exact results 1000x faster using index structures like HNSW and IVF.
✓
Hybrid Search: Keyword + Semantic
BM25 (keyword) finds exact matches. Semantic finds conceptual matches. Combining both gives best results. Used in production by Elasticsearch, Weaviate.
✓
Re-ranking
After semantic search retrieves top-50 candidates, a cross-encoder re-ranks them with higher accuracy. Slower but more precise final results.
▶ Semantic Search on Hugging Face Spaces
✓
Vector Databases (6 subtopics)
Specialized databases built to store, index, and search millions of vectors at speed. Essential for any production RAG system.
tool
✓
FAISS (Meta) — Local, Fast
Free, runs on your machine. Best for prototyping and offline use. Not a full database — just an index library. Great for <1M vectors.
✓
Chroma — Simple, Local or Cloud
The easiest vector DB to get started with. Works locally, no server needed. Has a simple Python API. Perfect for learning RAG.
✓
Pinecone — Managed Cloud
Serverless vector database. No infrastructure management. Free tier available. Production-ready. Best choice for side projects.
✓
Weaviate — Open Source + Hybrid Search
Open source, supports hybrid (vector + keyword) search natively. Can run locally or in cloud. Includes built-in modules for embedding.
✓
Qdrant — High Performance
Written in Rust. Very fast. Supports payload filtering (combine vector search with metadata filters). Growing fast in production use.
✓
pgvector — Postgres Extension
Add vector search to your existing Postgres database. If you already use Postgres, this is the simplest path. Less scalable but very practical.

Phase 4: Retrieval Augmented Generation (RAG)
Giving AI access to your private knowledge
0%
▼
What is this? RAG = Search + Generate. Instead of relying on what the model was trained on, you retrieve relevant information from your own documents and inject it into the prompt. The model then answers based on your data.

Why does it matter? LLMs hallucinate because they make things up when they don't know. RAG gives them facts to work from — eliminating most hallucinations for knowledge tasks.
Core Topics
✓
RAG Architecture (End-to-End) (4 subtopics)
The full pipeline: Ingest → Chunk → Embed → Store (offline). Then: Query → Embed → Search → Retrieve → Generate (online).
concept
✓
Naive RAG (Basic Pipeline)
The simplest version: chunk documents → embed → store in vector DB → on query, retrieve top-K chunks → stuff into prompt → get answer.
✓
Advanced RAG (Production-Grade)
Adds: query rewriting, hypothetical document embeddings (HyDE), hybrid search, re-ranking, metadata filtering, answer grounding checks.
✓
Modular RAG (Latest Research)
Treat each RAG component as a swappable module. Can add routing (choose which index to search), fusion (merge results from multiple sources), and iterative retrieval.
✓
Evaluation: RAGAS Framework
RAGAS measures: context precision (did we retrieve the right stuff?), context recall (did we retrieve all the right stuff?), faithfulness (is the answer grounded in context?), answer relevancy.
▶ RAG in 5 Min: LangChain Quickstart
✓
Document Chunking (5 subtopics)
LLMs have limited context windows, so you can't feed a whole book. You must split documents into chunks. How you split dramatically affects retrieval quality.
concept
✓
Fixed-Size Chunking
Split every N characters/tokens. Simple but can split mid-sentence. Add overlap (e.g., 20% overlap) to preserve context across boundaries.
✓
Sentence/Paragraph Chunking
Split at natural boundaries (periods, newlines). Preserves semantic completeness. Better quality than fixed-size for most documents.
✓
Semantic Chunking
Embed every sentence, then split where cosine similarity between consecutive sentences drops sharply. Finds natural topic boundaries. Best quality, slowest.
✓
Parent-Child Chunking
Store large chunks (parent) for context but embed small chunks (child) for precise retrieval. Retrieve small, return parent. Best of both worlds.
✓
Document Hierarchy (RAPTOR)
Recursively summarize chunks into higher-level summaries. Build a tree structure. Query at any level. Best for very long documents.
▶ LangChain Text Splitters
✓
Retrieval Strategies (6 subtopics)
Simple vector search is just the start. Better retrieval = better answers. These strategies dramatically improve what gets retrieved.
concept
✓
Dense Retrieval (Semantic)
Embed query → find nearest vectors. Good for concept-based queries. Misses exact keyword matches.
✓
Sparse Retrieval (BM25 / TF-IDF)
Classic keyword-based. Fast. Catches exact terms. Misses paraphrases. Use BM25 for technical queries with specific terms (error codes, names).
✓
Hybrid Retrieval + RRF
Run both dense and sparse, merge results with Reciprocal Rank Fusion (RRF). Consistently better than either alone. Standard in production.
✓
Query Expansion and HyDE
HyDE: generate a hypothetical perfect answer, embed it, use that for retrieval instead of the raw query. Improves recall for vague queries.
✓
Multi-Query Retrieval
Generate 3-5 variations of the user query, retrieve for each, merge unique results. Compensates for imperfect query phrasing.
✓
Self-Query Retrieval (Metadata Filtering)
Parse the query to extract filters ('show me papers from 2023 about attention'). Apply metadata filters alongside vector search for precision.
✓
Context Management and Injection (4 subtopics)
Getting relevant chunks is half the job. The other half is how you put them into the prompt without confusing the model.
concept
✓
Context Window Management
Calculate token budgets: system prompt + retrieved context + conversation history + response = must fit in context window. Use tiktoken to count.
✓
Prompt Templates for RAG
'Use ONLY the following context to answer. If the answer is not in context, say I don't know. Context: {context}. Question: {question}' — this prevents hallucination.
✓
Lost in the Middle Problem
LLMs underperform on info in the middle of long contexts. Put most important context at the start and end of your retrieved passages.
✓
Source Citations
Ask the model to cite which source each fact came from. Dramatically increases trustworthiness and lets users verify answers.
✓
Hallucination Reduction (4 subtopics)
LLMs will confidently make things up. These techniques ground the model in actual retrieved facts and catch lies.
concept
✓
Grounding Prompts
'Answer strictly based on the provided context. If uncertain, say so.' Clear instructions dramatically reduce hallucinations.
✓
Faithfulness Checking (RAGAS)
After generating, check each sentence in the answer against the retrieved context. Flag sentences with no supporting evidence.
✓
Confidence Scoring
Ask the model 'How confident are you on a scale of 1-10 and why?' Low-confidence answers can be flagged for human review.
✓
Chain-of-Verification (CoVe)
After generating an answer, generate verification questions, answer them independently from context, compare to original answer. Catches internal contradictions.
▶ RAGAS Evaluation Framework
✓
LangChain and LlamaIndex (4 subtopics)
The two main frameworks for building RAG systems. LangChain = general AI orchestration. LlamaIndex = specialized for document intelligence and RAG.
tool
✓
LangChain Core Concepts
Chains, LLMs, Prompts, Retrievers, Memory, Agents. LCEL (LangChain Expression Language) lets you compose pipelines with | operators.
✓
LlamaIndex vs LangChain
LlamaIndex: optimized for indexing, retrieval, query engines over documents. LangChain: more general, better for agents and complex workflows. Use LlamaIndex for pure RAG.

Building a RAG Chain in LangChain
load_documents → split → embed → vectorstore → retriever → RetrievalQA chain. 15 lines of code. Start here.

Phase 5: AI System Engineering
Building AI systems that work in the real world
0%
▼
What is this? Moving from 'it works on my machine' to production. This phase covers the engineering challenges of deploying AI at scale — safety, memory, performance, cost, and quality control.

Why does it matter? AI demos are easy. Production AI is hard. Every engineer who ships AI must know guardrails, caching, evaluation, and memory patterns.
Core Topics
✓
Memory in AI Systems (5 subtopics)
By default, LLMs have no memory between conversations. You must engineer memory — what to remember, how long, and how to retrieve it.
concept
✓
In-Context Memory (Conversation History)
Simply pass all previous messages in each API call. Works for short conversations. Expensive and hits context limits for long sessions.
✓
External Memory (Vector Store Memory)
Summarize or embed old messages, store in a vector DB, retrieve relevant past context on each turn. Scalable for long-term memory.
✓
Entity Memory
Extract key entities (person names, facts) from conversation and maintain a structured knowledge base about them. 'Bob mentioned he's a vegetarian.'
✓
Summary Memory
Periodically summarize conversation history into a compressed summary. Replace old messages with summary. Keeps context short.
✓
Episodic vs Semantic Memory
Episodic: specific events ('We discussed project X on Monday'). Semantic: general facts ('User prefers Python'). Both types needed for long-term agents.
✓
Guardrails and Safety (5 subtopics)
AI systems can produce harmful, incorrect, or off-topic outputs. Guardrails are filters that validate inputs and outputs to keep AI safe and on-task.
tool
✓
Input Guardrails (Prompt Injection Defense)
Detect and block attempts to override system instructions. Sanitize user input. Validate that input is on-topic before sending to LLM.
✓
Output Guardrails (Content Filtering)
Check outputs for: PII leakage, toxic content, hallucinations, off-topic responses, format violations. Block or reroute bad outputs.
✓
Guardrails AI Library
Python library that defines validators (no_profanity, valid_json, correct_format, etc.) and wraps LLM calls. Retries on failure.
✓
NeMo Guardrails (NVIDIA)
Define conversation rails in a declarative language (Colang). Block certain topics, enforce response templates, prevent off-topic dialogue.
✓
Custom Policy Enforcement
Use an LLM-as-judge to evaluate outputs against your business policies before returning to user. More flexible but adds latency.
▶ Guardrails AI Quickstart
✓
LLM Caching (4 subtopics)
LLM API calls are slow and expensive. Caching identical or similar requests saves money and reduces latency dramatically.
concept
✓
Exact Match Caching (Redis)
Hash the prompt → check Redis → return cached response if hit. Works for identical repeated queries. 0ms latency on hits.
✓
Semantic Caching (GPTCache)
Embed the query → check if semantically similar query was answered before → return cached answer if match found. Saves cost on paraphrased repeats.
✓
Anthropic and OpenAI Prompt Caching
Both APIs offer server-side caching for long system prompts. Reusing the same system prompt = 80-90% cost reduction on cached tokens. Enable it explicitly.
✓
Response Streaming and Partial Caching
Cache only the first N tokens if they're deterministic. Serve cached prefix, generate the rest live. Reduces both latency and cost.
✓
Streaming Responses (4 subtopics)
Instead of waiting 10 seconds for a full response, stream tokens as they're generated. This feels 10x faster to users even if total time is the same.
concept
✓
Server-Sent Events (SSE)
The technology behind streaming. Server sends events to client incrementally. Used by OpenAI, Anthropic APIs. Works over HTTP.
✓
Streaming in Python with OpenAI API
Set stream=True. Iterate over stream.choices[0].delta.content chunks. Each chunk is a token or few tokens. Simple to implement.
✓
Streaming in Web UIs (React / Vanilla JS)
Use fetch with ReadableStream or EventSource. Parse SSE events. Update DOM incrementally. This is how ChatGPT interface works.
✓
Handling Streaming Errors and Interruptions
Users may disconnect mid-stream. Always handle partial completions gracefully. Track streaming state and implement reconnection logic.
▶ OpenAI Streaming Guide
✓
Evaluation Pipelines (5 subtopics)
You can't improve what you don't measure. Build automated evaluation so you know when your AI is getting better or worse.
tool
✓
Building a Golden Dataset
Collect 50-200 real examples with ground-truth answers. This is your benchmark. Every model/prompt change is evaluated against this set.
✓
Automated Metrics (BLEU, ROUGE, Exact Match)
BLEU/ROUGE: measure word overlap with reference. Exact match: binary. Good for factual QA. Poor for generative tasks where many answers are valid.
✓
LLM-as-Judge Evaluation
Use GPT-4/Claude to grade outputs on rubrics (accuracy: 0-5, helpfulness: 0-5, safety: pass/fail). Correlates well with human judgment at scale.
✓
Tracing with LangSmith / Arize
Log every LLM call: inputs, outputs, latency, cost, token count. Enables debugging, regression testing, and performance monitoring.
✓
A/B Testing Prompts and Models
Route 50% of traffic to variant A, 50% to B. Track success metrics (user rating, task completion). Ship the winner.
▶ LangSmith Tracing
✓
Prompt Templates and Management (4 subtopics)
Prompts are code. They need version control, templating, and management — especially when you have hundreds of prompts in production.
tool
✓
Jinja2 Templating for Prompts
Use Jinja2 (Python) to create prompt templates with variables: {{ user_name }}, {% if has_context %} ... {% endif %}. Dynamic, reusable prompts.
✓

06
Phase 6: Agentic AI
AI that plans, acts, and loops until the job is done
0%
▼
What is this? AI agents are LLMs that can use tools (search, code execution, APIs), plan multi-step tasks, and loop until the goal is achieved — all without human guidance for each step.

Why does it matter? This is the frontier of AI engineering. Agents will automate entire workflows. Understanding agents now puts you 2 years ahead of most developers.
Core Topics
✓
What is an AI Agent? (4 subtopics)
An AI that can decide what to do next, use tools to do it, observe results, and repeat until the task is complete. Like an employee who can use a computer.
concept
✓
The ReAct Loop: Thought → Action → Observation
Thought: 'I need to search for X.' Action: call search tool. Observation: read result. Thought: 'Now I should summarize.' This loop is the heart of all agents.
✓
Agents vs Chains vs Pipelines
Pipeline: fixed steps A→B→C. Chain: dynamic but linear. Agent: dynamic, can loop, branch, choose tools, and decide when to stop. Agents have true agency.
✓
Planning: Zero-Shot vs Plan-and-Execute
Zero-shot: decide next action at each step (flexible but can get lost). Plan-and-Execute: generate full plan first, then execute. Better for complex multi-step tasks.
✓
Tool Calling (Function Calling)
Pass a list of tool schemas to the LLM. It outputs a structured JSON call (tool name + arguments). Your code executes it. This is the standardized agent protocol now.
▶ Build an Agent: LangGraph Quickstart
✓
Tool Design and Integration (5 subtopics)
Agents are only as powerful as their tools. Well-designed tools with clear descriptions let the agent use them correctly without hallucinating.
tool
✓
Core Tool Types: Search, Code, API, DB
Web search (Tavily, Serper). Code execution (Python REPL, E2B sandbox). API calls (weather, databases). File operations. Email/calendar.
✓
Tool Schema Design (Name, Description, Parameters)
LLM reads your tool description to decide when to call it. Bad description = wrong calls. Be very specific: 'Search the web for real-time information. Do NOT use for general knowledge.'
✓
Tool Error Handling
Tools fail. Always return structured errors the LLM can read: {error: 'API rate limit exceeded', suggestion: 'Wait 60 seconds and retry'}. The agent can then self-recover.
✓
Sandboxed Code Execution (E2B)
Let AI write and run code in an isolated sandbox. E2B provides cloud sandboxes. Code Interpreter feature in ChatGPT uses this. Critical for code-generating agents.
✓
MCP (Model Context Protocol)
Anthropic's open standard for connecting AI models to external tools and data. Write an MCP server once and any compatible agent can use it. The future of tool integration.
▶ E2B Code Sandbox
✓
Memory Systems for Agents (4 subtopics)
Long-running agents need to remember what they did, what they learned, and user preferences across sessions.
concept
✓
Working Memory (In-Context)
The current conversation and tool results. Limited to context window. Fast. Used for the current task only.
✓
Long-Term Memory (Vector Store)
Persist important information across sessions. Embed and store key facts, user preferences, task summaries. Retrieve on next session.
✓
Episodic Memory (Past Actions Log)
Log of what the agent did and what worked. Enables learning from experience: 'Last time I tried X and it failed. I'll try Y instead.'
✓
Shared Memory in Multi-Agent Systems
Multiple agents share a common knowledge store. Agent A's findings become available to Agent B. Enables true collaboration.
✓
LangGraph — Stateful Agent Framework (5 subtopics)
LangGraph lets you build agents as explicit state machines with nodes and edges. Much more controllable than simple ReAct agents. The current industry standard.
tool
✓
Graphs, Nodes, and Edges
Node = a function that processes state (call LLM, call tool, check condition). Edge = the transition between nodes. Conditional edges = branches.
✓
State Management
Each graph has a state schema (TypedDict). Every node reads and updates the state. This makes complex agent logic debuggable and testable.
✓
Human-in-the-Loop with Interrupts
LangGraph can pause before any node and wait for human approval. Use this for: sending emails, making payments, deleting data.
✓
Persistence and Checkpointing
Save graph state to database after each step. Resume from any checkpoint. Enables long-running agents, retries, and time travel debugging.
✓
Streaming Agent Events
Stream not just tokens but graph events: which node is executing, tool call inputs/outputs, current state. Full observability.
▶ LangGraph Tutorial
✓
Multi-Agent Systems (5 subtopics)
Some tasks are too complex for one agent. Split into specialized agents that collaborate — each expert in one domain.
tool
✓
Orchestrator-Subagent Pattern
One orchestrator agent breaks the task, delegates subtasks to specialist agents (research agent, writing agent, code agent), aggregates results.
✓
CrewAI — Role-Based Agents
Define agents with roles, goals, and backstories. Assign tasks. Agents collaborate with defined communication patterns. Great for business workflows.
✓
AutoGen — Conversational Multi-Agent
Microsoft's framework. Agents communicate via messages. Can include human proxy agents. Code-focused. Good for software engineering tasks.
✓
Agent Communication Protocols
Agents can communicate via: direct function calls (fast), shared message queue (async), shared memory (concurrent). Choose based on your latency needs.
✓
Avoiding Agent Loops and Hallucinated Actions
Set max_iterations limits. Validate tool call arguments before execution. Add observation quality checks. Agents can spiral if unconstrained.
▶ CrewAI Quickstart
✓
Agent Evaluation and Observability (4 subtopics)
Agents fail in unpredictable ways. You need full visibility into every decision, tool call, and reasoning step to debug and improve them.
tool
✓
Tracing with LangSmith
Every LangGraph/LangChain run logged automatically: full trace of LLM calls, tool calls, latency, token counts, errors. Essential for production agents.
✓
AgentOps — Agent Monitoring
Purpose-built for agents. Tracks agent sessions, tool usage patterns, success rates, costs. Identify which tools fail most often.
✓
Evaluation Metrics for Agents
Task completion rate, number of steps to completion, tool use accuracy, hallucination rate in tool calls, cost per task. Track all of these.
✓
Adversarial Testing (Red Teaming)
Actively try to break your agent: prompt injection in tool results, misleading search results, impossible tasks. Agents are fragile to adversarial inputs.
